# Руководство по внесению вклада

Спасибо за интерес к проекту "Планировщик ремонта"! Мы приветствуем вклад от сообщества.

## Как внести вклад

### 1. Сообщить об ошибке
- Используйте GitHub Issues
- Опишите проблему детально
- Приложите скриншоты если возможно
- Укажите версию браузера и ОС

### 2. Предложить улучшение
- Создайте Issue с меткой "enhancement"
- Опишите предлагаемую функциональность
- Объясните почему это полезно

### 3. Отправить Pull Request
1. Форкните репозиторий
2. Создайте ветку для вашей фичи (`git checkout -b feature/AmazingFeature`)
3. Сделайте изменения
4. Закоммитьте (`git commit -m 'Add some AmazingFeature'`)
5. Запушьте (`git push origin feature/AmazingFeature`)
6. Откройте Pull Request

## Стандарты кода

### Python (Backend)
- Следуйте PEP 8
- Используйте type hints
- Добавляйте docstrings
- Пишите тесты

### TypeScript (Frontend)
- Используйте ESLint
- Следуйте airbnb style guide
- Пишите типобезопасный код
- Добавляйте комментарии для сложной логики

## Структура коммитов

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Types:**
- feat: новая функциональность
- fix: исправление бага
- docs: изменения в документации
- style: форматирование, отступы
- refactor: рефакторинг кода
- test: добавление тестов
- chore: обновление зависимостей и т.д.

**Примеры:**
```
feat(editor): add drag and drop for walls
fix(api): correct validation for gas equipment
docs(readme): update installation instructions
```

## Тестирование

Перед отправкой PR убедитесь, что:
- [ ] Код проходит линтеры
- [ ] Все тесты проходят
- [ ] Добавлены тесты для новой функциональности
- [ ] Документация обновлена
